# treasury-frontoffice

treasury agent

Generated by `atom create`. Uses atom-sdk (agentscope) with a ReAct agent loop.
All LLM calls route through GATE — authenticated, rate-limited, and audit-logged.

---

## Setup

### Step 1 — Install dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install "git+https://github.com/shreyasY2k/atom.git#subdirectory=atom-sdk/atom_platform_sdk"
```

### Step 2 — Activate the virtual env in new terminals

```bash
source .venv/bin/activate
```

### Step 3 — Create agent in atom-studio

```
Open http://localhost:3000 (docker) or http://studio.atom.local (k8s)

1. Domains → New Domain → give it a name
2. Agents  → New Agent  → fill wizard (pick gemini-2.5-flash in allowed models)
3. Copy the one-time JWT token shown at the end
4. Note domain UUID and agent UUID from the Studio URL:
     http://studio.atom.local/domains/<DOMAIN-ID>/agents/<AGENT-ID>
```

### Step 4 — Configure .env

```bash
# .env was created from .env.example — fill in the values:
ATOM_GATE_URL=http://localhost:8080    # or http://gate.atom.local (k8s)
ATOM_DOMAIN_ID=<domain-uuid>
ATOM_AGENT_ID=<agent-uuid>
ATOM_AGENT_JWT=<token-from-studio>
ATOM_MODEL_NAME=gemini-2.5-flash
```

### Step 5 — Run

```bash
python agent.py
```

## Deploy to ATOM

Once the agent runs locally, deploy it with full governance:

```bash
bin/atom deploy
# Submits for HITL approval in atom-studio → atom-runtime deploys the pod
```

Approve in Studio → HITL queue. Then call the deployed agent through GATE:

```bash
curl -X POST http://localhost:8080/domain/<domain-id>/agent/<agent-id>/run \
  -H "Authorization: Bearer <agent-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello!"}'
```
